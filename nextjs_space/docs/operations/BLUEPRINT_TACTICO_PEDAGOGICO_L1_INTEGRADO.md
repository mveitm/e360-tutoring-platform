# Blueprint Tactico-Pedagogico PAES Competencia Lectora L1 - Bexauri

## 1. Resumen ejecutivo

Este documento define el Blueprint Tactico-Pedagogico PAES_L1 integrado para Bexauri. Su proposito es documentar la derivacion especifica decidida para la linea L1, sin implementar contenido nuevo, sin modificar registry y sin declarar aprobacion de estudiante o producto.

PAES_L1 no debe tratarse como un banco de textos, una coleccion de preguntas de lectura ni una promesa de preparacion PAES. Debe existir como tutoria Bexauri test-oriented, centrada en Competencia Lectora, con proposito, texto visible, tarea focalizada, evidencia interpretable, feedback, autorreporte, revision y continuidad pedagogicamente gobernada.

Resultado rector:

```text
PAES_L1_TACTICAL_PEDAGOGICAL_BLUEPRINT_DERIVED
```

## 2. Relacion con BPCPI, BPTT y documentacion L1 existente

PAES_L1 hereda del BPCPI el principio de que actividad no equivale a aprendizaje y que toda evidencia tiene limites. Hereda del BPTT la gramatica transversal de tutoria:

```text
proposito
-> dominio/habilidad
-> progresion
-> recurso
-> capsula
-> evidencia
-> feedback
-> autorreporte
-> continuidad
-> progreso
-> revision
```

El piloto L1 existente en registry se considera un artefacto interno candidato, no una autorizacion de uso de estudiante o producto.

## 3. Rol del Gerente Pedagogico L1

El Gerente Pedagogico L1 es la autoridad conceptual que traduce BPCPI y BPTT al dominio PAES Competencia Lectora. Debe gobernar foco lector, seleccion o creacion de textos, derechos, tareas, evidencia, feedback, autorreporte, revision, continuidad y limites de inferencia.

Debe rechazar textos sin trazabilidad de fuente/derechos, tareas que no dependan del texto visible, feedback generico, claims de diagnostico, continuidad automatica sin evidencia suficiente y cualquier declaracion de uso de producto no respaldada por metadata vigente.

## 4. Definicion de la tutoria PAES_L1 en Bexauri

PAES_L1 es una experiencia guiada para desarrollar destrezas de Competencia Lectora en contexto PAES mediante textos o recursos de lectura, preguntas focalizadas, evidencia de respuesta, feedback accionable, revision y continuidad cautelosa.

No es una preparacion completa, un simulador oficial, una promesa de puntaje, una certificacion de dominio, una ruta adaptativa inteligente ni un set productivo aprobado por el solo hecho de estar registrado.

## 5. Principio rector operativo L1

Cada capsula L1 debe convertir una interaccion lectora breve en evidencia interpretable, feedback util y una decision de continuidad pedagogicamente justificable.

Cadena minima:

```text
proposito lector
-> texto visible
-> tarea
-> evidencia
-> feedback
-> autorreporte
-> revision
-> continuidad
```

## 6. Que significa aprender L1 en Bexauri

Aprender L1 significa mejorar la capacidad de localizar, seleccionar, interpretar y usar informacion textual con foco, precision y revision. Una respuesta correcta aislada no autoriza inferencias fuertes sobre dominio lector.

En L1, la evidencia debe conectarse con lo que el estudiante efectivamente leyo y respondio. El feedback debe ayudar a volver al texto, distinguir dato explicito de distractor y reconocer la estrategia lectora usada.

## 7. Familias/ejes iniciales de destrezas L1

Familias iniciales:

1. Localizacion de informacion explicita.
2. Identificacion de condiciones, tiempos, lugares y actores.
3. Relacion entre informacion literal y opcion de respuesta.
4. Distincion entre dato solicitado, dato cercano y distractor.
5. Revision de evidencia textual antes de responder.

Estas familias no declaran cobertura completa de Competencia Lectora.

## 8. Tipos de capsulas L1

L1 puede usar capsulas de localizacion, seleccion de informacion, comprension literal, relacion dato-opcion, revision de distractores y refuerzo focalizado.

Las capsulas deben ser breves, cerrables y trazables a un texto o recurso visible. El maximo recomendado para el estandar inicial Bexauri es 1 a 4 preguntas por capsula, salvo documento posterior autorizado.

## 9. Recurso textual / reading asset

Todo recurso textual L1 debe declarar al menos:

- identificador de texto;
- version;
- titulo;
- fuente o clase de fuente;
- base de derechos;
- dependencia oficial, si existe;
- dependencia copyrighted, si existe;
- estado de revision.

Si el texto no tiene derechos auditables, no debe avanzar a flujo de estudiante/producto.

## 10. Evidencia pedagogica L1

Evidencia minima:

- opcion elegida;
- acierto/error;
- item y texto asociado;
- distractor elegido cuando aplique;
- revision de feedback;
- autorreporte si existe;
- completitud de capsula.

Esta evidencia permite hipotesis debiles sobre localizacion de informacion y uso de datos textuales. No permite diagnostico definitivo, puntaje, dominio estable ni preparacion completa.

## 11. Feedback L1

El feedback breve debe indicar el dato textual o criterio lector central sin reemplazar la lectura. El feedback completo debe restituir la pregunta, ubicar la evidencia en el texto, justificar la opcion correcta y explicar por que los distractores no responden la tarea.

El feedback no debe afirmar que el estudiante domina una habilidad ni prometer avance automatico.

## 12. Autorreporte y metacognicion L1

L1 debe incorporar o preparar autorreporte sobre dificultad percibida, seguridad de respuesta, necesidad de volver al texto y claridad del feedback. En ausencia de autorreporte implementado, la continuidad debe ser conservadora.

## 13. Continuidad pedagogica L1

La continuidad L1 debe basarse en evidencia acumulada y documentada. Con un solo set piloto, la continuidad automatica queda bloqueada si la metadata vigente asi lo indica.

La continuidad puede proponer revision o siguiente set solo cuando existan guardrails, segundo set revisado si corresponde y aprobacion documental suficiente.

## 14. Roadmap maestro L1 y microroadmap vivo

La rama L1 distingue:

```text
secuencia simple
roadmap maestro L1
microroadmap vivo del estudiante
```

El set piloto actual no declara roadmap maestro completo ni microroadmap vivo. Cualquier roadmap maestro L1 futuro requiere familias, nodos, prerrequisitos, tipos de texto, tipos de tarea, criterios de avance/refuerzo/revision y politica de derechos.

## 15. Derechos, fuentes y revision de textos

Todo texto L1 debe ser original, autorizado o contar con derechos documentados. Debe indicarse si hay dependencia oficial o copyrighted. El uso de textos externos sin revision de derechos queda bloqueado.

El estado actual del set piloto declara fuente original Bexauri candidata y dependencia copyrighted `none`, pero su aprobacion sigue limitada a candidato interno segun approvalMetadata vigente.

## 16. Limites de inferencia y claims prohibidos

Permitido:

- practica guiada L1;
- actividad completada;
- feedback disponible;
- evidencia registrada;
- revision disponible.

Cauteloso:

- avance;
- progreso;
- refuerzo;
- siguiente paso.

Prohibido:

- puntaje probable;
- dominio;
- diagnostico automatico;
- cobertura completa;
- preparacion PAES completa;
- uso productivo aprobado sin metadata;
- continuidad automatica sin contrato;
- equivalencia con prueba oficial.

## 17. Student-use y product-use gates

La aprobacion de estudiante y producto debe gobernarse por metadata explicita. Mientras `studentUseApproved=false` y `productUseApproved=false`, L1 no debe declararse aprobado para uso de estudiante/producto.

Los gates deben separar revision pedagogica, revision de derechos, aprobacion de uso, acceso local/dev y smoke humano.

## 18. Relacion con el set piloto actual

El set vigente es:

```text
title: PAES L1 - Localizacion de informacion - Piloto interno 01
contentKey: l1_locating_information_pilot_set_01
contentType: reading_l1_locating_information
items: 3
studentUseApproved: false
productUseApproved: false
```

Este set puede servir como artefacto interno para gobernanza y revision, no como aprobacion de producto.

## 19. Criterios de aceptacion de capsula L1

Una capsula L1 debe tener:

- foco lector explicito;
- texto visible y trazable;
- fuente/derechos documentados;
- items alineados al texto;
- `correctOptionKey` por item;
- feedback breve y completo segun patron vigente;
- ausencia de filtracion indebida;
- legibilidad movil;
- limites de claims visibles;
- continuidad gobernada.

## 20. Criterios de rechazo

Rechazar capsulas L1 con texto sin derechos, item que no se responde desde el texto, clave ambigua, feedback generico, distractores no trazables, inferencias excesivas, claims de puntaje/dominio, continuidad automatica no autorizada o aprobacion de uso no documentada.

## 21. Decisiones que deben escalarse

Escalar:

- cambio de approvalMetadata;
- uso de texto externo o oficial;
- habilitacion de uso de estudiante/producto;
- continuidad automatica;
- declaracion de roadmap maestro;
- segundo set revisado para continuidad;
- cualquier claim comercial o de cobertura.
