# FL-CONTENT-1 — First Real PAES M1 Beta Task

## 1. Título

**PAES M1 — Ecuaciones lineales básicas**

---

## 2. Propósito

Esta es la primera tarea pedagógica real mínima para PAES M1 dentro del beta cerrado asistido de Bexauri.

Su objetivo **no** es medir mejora de puntaje PAES, sino validar que:
- La experiencia pedagógica es clara y útil.
- El estudiante puede trabajar de forma autónoma.
- El ciclo evidencia → revisión → decisión funciona con contenido real.

---

## 3. Instrucción para el estudiante

> **Actividad: Ecuaciones lineales básicas**
>
> A continuación encontrarás 8 ejercicios de selección múltiple sobre ecuaciones lineales.
>
> **¿Qué hacer?**
> 1. Lee cada ejercicio con calma.
> 2. Resuélvelo en tu cuaderno o en papel.
> 3. Anota la alternativa que elegiste para cada ejercicio.
> 4. Cuando termines todos los ejercicios, vuelve a Bexauri.
> 5. Presiona **«Terminar»** y cuéntanos cómo te fue.
>
> **Tiempo estimado:** 20–30 minutos.
>
> No hay presión por hacerlo perfecto. Lo importante es que lo intentes y nos cuentes cómo te fue.

### Ejercicio 8

| | |

## 5. Pauta de respuestas

| Ejercicio | Respuesta correcta | Resolución breve |
|---|---|---|
| 1 | **C) 5** | $3x + 5 = 20 \Rightarrow 3x = 15 \Rightarrow x = 5$ |
| 2 | **D) 11** | $2x - 7 = 15 \Rightarrow 2x = 22 \Rightarrow x = 11$ |
| 3 | **C) 6** | $5x = 3x + 12 \Rightarrow 2x = 12 \Rightarrow x = 6$ |
| 4 | **C) 7** | $4(x-2) = 20 \Rightarrow 4x - 8 = 20 \Rightarrow 4x = 28 \Rightarrow x = 7$ |
| 5 | **C) 18** | $\frac{x}{3} + 4 = 10 \Rightarrow \frac{x}{3} = 6 \Rightarrow x = 18$ |
| 6 | **C) 16** | $x + 9 = 25 \Rightarrow x = 16$ |
| 7 | **C) 10** | $3x - 6 = 24 \Rightarrow 3x = 30 \Rightarrow x = 10$ |
| 8 | **B) $3x + 500 = 3.200$** | El precio de 3 cuadernos ($3x$) más $500 es $3.200 |

## 6. Criterios de revisión admin

| Decisión | Cuándo aplicar |
|---|---|
| **advance** | El estudiante completó la tarea y reporta claridad y/o confianza. La evidencia sugiere que está listo para un nuevo tema o mayor dificultad. |
| **reinforce** | El estudiante reporta dificultad o comprensión débil. El mismo tema debe revisarse con un enfoque diferente. |
| **hold** | La evidencia es insuficiente para decidir. Se necesita más información o un nuevo intento. |
| **redirect** | La tarea resultó confusa o no coincide con el nivel del estudiante. Se necesita un tema o enfoque diferente. |

La decisión siempre es manual y humana. No se usa corrección automática ni IA.

---

## 7. Rationales sugeridos

**Si el estudiante reporta "Me fue bien":**
> "El estudiante completó la práctica de ecuaciones lineales y reportó sentirse bien. La evidencia sugiere comprensión básica del tema. Se recomienda avanzar a un tema nuevo o a mayor dificultad dentro de álgebra."

**Si el estudiante reporta "Me costó":**
> "El estudiante completó la práctica pero reportó dificultad. Se recomienda reforzar ecuaciones lineales con ejercicios más guiados o con un enfoque paso a paso antes de avanzar."

**Si la evidencia es insuficiente:**
> "El estudiante completó la carga pero el autorreporte no ofrece información suficiente para evaluar comprensión. Se recomienda mantener el tema y solicitar un reporte más detallado en la próxima sesión."

**Si la tarea parece no coincidir con el nivel:**
> "El estudiante reportó confusión con la tarea. Se recomienda redirigir a un tema introductorio o a una actividad de nivelación antes de volver a ecuaciones lineales."

## 9. Cómo usar esta tarea en el MVP actual

**En la plataforma:**
- **Título del StudyLoad:** `PAES M1 — Ecuaciones lineales básicas`
- **loadType:** `practice`
- **status:** `pending`

**Entrega del contenido:**
Actualmente, el contenido de los ejercicios se entrega **externamente** al estudiante:
- Por WhatsApp (texto o imagen)
- Por Google Docs (enlace compartido)
- Por PDF
- Por mensaje directo

El estudiante trabaja los ejercicios fuera de la plataforma y luego regresa a `/now` para presionar **Terminar** y dejar su autorreporte.

**Flujo completo:**
1. Admin crea el StudyLoad en la plataforma (pending).
2. Admin envía los ejercicios al estudiante por canal externo.
3. Estudiante entra a `/now`, presiona **Empezar**.
4. Estudiante trabaja los ejercicios externamente.
5. Estudiante regresa a `/now`, presiona **Terminar**.
6. Estudiante escribe su autorreporte.
7. Admin revisa evidencia en el detalle del ciclo.
8. Admin crea una CycleDecision con rationale.

---

## 10. Qué valida esta tarea

- Claridad de la instrucción para el estudiante
- Capacidad del estudiante para trabajar de forma autónoma
- Calidad del autorreporte como señal pedagógica
- Capacidad del admin para revisar evidencia y tomar una decisión manual
- Primera señal real de utilidad pedagógica del ciclo completo
- Viabilidad del flujo: contenido externo → plataforma → evidencia → decisión

---

## 11. Qué NO valida esta tarea

- **No valida mejora de puntaje PAES.** Es una sola práctica inicial.
- **No valida preparación PAES completa.** Cubre solo ecuaciones lineales básicas.
- **No valida corrección automática.** No hay auto-corrección en la plataforma.
- **No valida tutoría adaptativa con IA.** Las decisiones son manuales.
- **No valida beta a gran escala.** Es para 1–2 participantes controlados.

## 13. Declaración explícita de no-implementación

FL-CONTENT-1 es una fase de documentación/contenido. **No**:

- Modifica código de la aplicación
- Modifica esquema de base de datos
- Modifica endpoints de API
- Despliega la aplicación
- Crea usuarios, estudiantes, matrículas, ciclos, cargas de estudio, respuestas o decisiones
- Ejecuta `prisma db push`, `prisma migrate`, o `prisma db seed`
- Inicia, completa, cierra o continúa ninguna carga o ciclo
- Crea agentes de IA o automatización
- Imprime secretos, contraseñas o cadenas de conexión

Todas las acciones descritas en este documento son recomendaciones para ejecución manual por el operador humano en fases posteriores.
