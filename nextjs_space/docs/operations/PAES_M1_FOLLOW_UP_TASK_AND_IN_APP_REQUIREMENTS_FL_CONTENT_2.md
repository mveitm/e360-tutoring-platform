# PAES M1 — Problemas con ecuaciones lineales

## 1. Title

**PAES M1 — Problemas con ecuaciones lineales**

## 2. Purpose

This is the second internal pedagogical task for the E360 / Bexauri self-beta, created after the successful first execution of "PAES M1 — Ecuaciones lineales básicas".

This task:

- follows the `advance` decision from the first capsule;
- increases difficulty slightly — from direct linear equations to equations with parentheses, fractions, and word problems requiring equation setup;
- keeps the task manageable within a single autonomous session;
- validates pedagogical continuity after a manual admin decision, not external beta readiness;
- helps define what future in-app interactive study loads must support.

## 7. Short solution notes

1. **2(x + 3) = 18** → 2x + 6 = 18 → 2x = 12 → x = 6.
2. **4x - 5 = 2x + 11** → 4x - 2x = 11 + 5 → 2x = 16 → x = 8.
3. **(x/2) + 7 = 15** → x/2 = 8 → x = 16.
4. **3(x - 4) + 6 = 24** → 3x - 12 + 6 = 24 → 3x - 6 = 24 → 3x = 30 → x = 10.
5. **x + 2x = 36** → 3x = 36 → x = 12. "Un número aumentado en el doble del mismo" significa x + 2x.
6. **x + 5 = 23** → x = 18. La edad actual más 5 años es 23.
7. **4x + 600 = 2.200** → 4x = 1.600 → x = 400. Cuatro lápices más la goma de $600.
8. La cuota fija es $2.000 y cada clase cuesta $500. El modelo es 2.000 + 500x = 5.500. La variable x multiplica el costo por clase, no la cuota fija.

## 8. Admin review criteria

### advance

Use if the student completes the task, reports "Me fue bien", and handles most equation setup and solving correctly — including at least some of the word problems and the modeling question.

### reinforce

Use if the student completes the task but reports difficulty with word problems, parentheses, or fractions. The student may need an additional practice with guided examples before moving to new content.

### hold

Use if evidence is incomplete — the student did not complete enough of the task, or the self-report and evidence are contradictory or insufficient to judge progress.

### redirect

Use if the main problem is reading comprehension, task mismatch, or the activity is clearly too difficult for the student's current level. Consider returning to simpler direct equations or a different skill area.

## 12. Why external students are still not invited

External students should **not** be invited yet because the platform still needs:

- **A beta-quality student frontend:** The current `/now` flow is functional but minimal — it does not explain context, instructions, or next steps clearly enough for an unassisted user.
- **In-app study load content:** Students currently work from external documents. A real beta experience requires content visible inside the app.
- **Interactive or guided task execution:** Multiple-choice exercises should be answerable within the app, not only on paper.
- **Clearer onboarding:** A new student should understand what Bexauri is, what PAES preparation means here, and what to expect — without WhatsApp or verbal explanation.
- **Stable admin-assisted continuity:** The admin loop works, but it depends heavily on manual coordination. Before inviting others, continuity after decisions should be smoother.
- **Reduced dependence on external documents or WhatsApp:** The platform should be the primary channel for content delivery and communication, not a secondary tracking tool.

## 13. Minimum in-app StudyLoad requirements before external beta

The following capabilities must exist in Bexauri before inviting external participants.

### Student side

- Open a StudyLoad and see full instructions inside the app.
- See estimated time for the task.
- See exercises inside the app (not only in an external document).
- Answer multiple-choice questions within the app.
- Save or submit responses.
- See progress through the task (e.g., "Exercise 3 of 8").
- Finish the task.
- Leave self-report.
- Understand what happens next (waiting for admin review, next load, etc.).

### Admin side

- See student responses (what the student answered per item).
- See score or completion summary if available.
- See self-report.
- See time and completion state if available.
- Create manual decision.
- Prepare and assign next load.

### Data model / future requirements

- **StudyLoad needs content or linked content:** The StudyLoad record should reference or contain the task instructions and exercises, not depend on an external document.
- **Responses need to be captured per item:** Each exercise answer should be stored individually, enabling item-level analysis.
- **Task versioning should exist:** If a task is updated, prior responses should still reference the original version the student saw.
- **Item-level evidence should become possible:** The admin should eventually see what the student answered per question, not only the self-report.
- **Admin should know what the student answered, not only how they felt:** Self-report alone is insufficient for informed decisions — actual responses provide the necessary evidence.

## 14. Proposed minimum interactive StudyLoad v1

A lightweight first version of in-app interactive study loads should include:

- **Title:** Displayed prominently when the student opens the load.
- **Instructions:** Clear, student-facing text explaining what to do and how long it should take.
- **5–8 multiple-choice items:** Each with a question stem and 4 options.
- **Answer capture:** The student selects one option per item; selections are saved.
- **Submit button:** The student explicitly submits their answers when done.
- **Self-report:** After submission, the student leaves their self-report as in the current flow.
- **Admin summary:** The admin can see which options the student selected, how many were correct (if an answer key exists), and the self-report.

What this version does **not** include:

- No automatic adaptation based on results.
- No AI agent or chatbot tutoring.
- No automatic scoring feedback to the student.
- No timed constraints.
- No rich media (images, diagrams, videos).
