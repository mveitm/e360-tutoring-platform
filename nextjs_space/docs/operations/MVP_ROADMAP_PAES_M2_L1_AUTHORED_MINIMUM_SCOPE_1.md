## Roadmap anchor

Current M1 repaired set has passed human regression. The next roadmap step is to define minimum authored scope for M2 and L1 before adding more capsules or opening productive agentic behavior.

This phase preserves the current controlled `MVP-Beta-Pre-Sales-Ready` posture. It does not declare Sales-Ready, M2 readiness, L1 readiness, full PAES preparation, autonomous diagnosis, or adaptive AI.

## Current state after M1 regression

Current state:

- M1-C01 through M1-C08 have passed simplified human regression for the recently repaired surfaces.
- M1 is the strongest operating reference, but it must not be copied blindly into M2 or L1.
- M2 and L1 are strategically required for the broader product promise, but current documentation still treats them as not ready for student-facing pilot claims.
- L1 requires a different StudyLoad shape because source text structure is part of the assessed skill.
- M2 likely requires more complex mathematical and visual-stimulus governance than current M1 seed capsules.
- Productive agentic remains blocked until authored/curated coverage, metadata, QA, smoke evidence and governance gates are stronger.

## Minimum authored scope principles

The minimum authored scope must demonstrate the product promise across M1/M2/L1 without implying full PAES preparation. The first closed pilot scope should be small enough to build and QA, but complete enough to test real student experience.

Principles:

- define the smallest credible M2/L1 authored seed;
- keep each capsule minimal, sufficient and closable;
- require brief and complete authoredFeedback from the first implementation batch;
- require visible, legible and auditable visual/text stimuli when the skill depends on them;
- require visible correlatives before human smoke;
- avoid internal leakage and commercial/agentic claims;
- separate scope/planning from implementation;
- keep continuity/route-order decisions in later explicit phases;
- keep M2/L1 readiness claims blocked until smoke/review evidence exists.

## PAES_M2 minimum scope

### Minimum to validate circuit

Recommended threshold:

```text
2-3 authored M2 capsules
```

Purpose:

- validate M2 StudyLoad content shape;
- validate M2 item count, feedback, submission and completed review compatibility;
- identify M2-specific notation, figure and graph/rendering needs;
- avoid any student-facing M2 readiness claim.

Recommended first coverage:

| capsule line | focus | exercise type | stimulus expectation |
| --- | --- | --- | --- |
| M2 seed 1 | algebra/functions/modelling | text-first problems with higher reasoning than M1 | plain text or rendered math only if ambiguity appears |
| M2 seed 2 | probability/statistics/data interpretation | interpretation/comparison from data | formatted table or graph plan if visual reading is central |
| M2 seed 3 | geometry/advanced reasoning | figure-dependent or relation-dependent reasoning | only if deterministic figure/approved visual path is specified |

### Minimum for first closed pilot exposure

Recommended threshold:

```text
4-8 authored M2 capsules
```

Required scope:

- at least two distinct mathematical areas, not only algebra;
- at least one data/probability/statistics capsule;
- at least one item requiring a visual stimulus classification;
- no geometry capsule enters pilot exposure without deterministic figure or approved static asset plan;
- all items comply with M2 maximum 4 exercises per capsule;
- all items have brief/complete authoredFeedback;
- all candidate capsules pass static review and human smoke/readiness.

### Minimum before agentic

Recommended threshold:

```text
12-20 curated authored M2 capsules
```

Required before any M2 agentic recommendation:

- stable metadata for axis/skill, difficulty, stimulus type, feedback status and smoke status;
- no unresolved P0 feedback/key/visual-stimulus blockers;
- enough diversity that recommendation is not choosing from a nominal set;
- human/QA approval of any visual stimuli.

### What remains outside first M2 scope

Out of scope:

- full PAES_M2 preparation claim;
- public M2 readiness claim;
- broad M2 bank generation;
- complex SVG/figure engine unless separately scoped;
- productive agentic recommendation/generation;
- route-order/continuity implementation;
- DB/schema work.

### M2 smoke/readiness criteria

Before human UI smoke:

- visible M2 correlativo defined;
- title/contentKey proposed in a later implementation plan;
- 3-4 exercises per capsule unless explicitly justified;
- correctOptionKey reviewed;
- brief/complete authoredFeedback present;
- any table/graph/figure/math stimulus classified;
- no internal leakage;
- no Sales-Ready/adaptive AI/mastery/theta/score claims.

## PAES_L1 minimum scope

### Minimum to validate circuit

Recommended threshold:

```text
1-2 authored L1 capsules
```

Purpose:

- validate text-base presentation;
- validate question grouping from one source text;
- validate feedback display and completed review for reading questions;
- avoid any L1 readiness claim.

Required first flow:

- at least one structured text-base capsule;
- source text visually separated from questions;
- 1-4 answerable questions;
- feedback that references the text or fragment;
- no screen overload from text plus questions;
- no internal rationale or metadata visible.

### Minimum for first closed pilot exposure

Recommended threshold:

```text
4-8 authored L1 capsules
```

Recommended coverage:

| capsule line | reading focus | text-base type | question shape |
| --- | --- | --- | --- |
| L1 seed 1 | localizar informacion explicita | short informational text | 2-4 direct evidence questions |
| L1 seed 2 | idea principal/proposito | short expository or informational text | 2-4 comprehension questions |
| L1 seed 3 | inferencia local | short narrative/informational fragment | 2-4 inference questions |
| L1 seed 4 | vocabulary/context or relation between ideas | short text with clear lexical/context cues | 2-4 context questions |

Optional extension to 8 capsules:

- add text variety while keeping each text short and reviewable;
- include at least one question using a clearly marked fragment or citation;
- include one mixed explicit/inference capsule only after simpler flows pass.

### Minimum before agentic

Recommended threshold:

```text
12-20 curated authored L1 text-base capsules
```

Required before any L1 agentic recommendation:

- stable metadata for text type, reading skill, difficulty, question count and feedback status;
- text-base QA and citation/fragment conventions;
- no unresolved leakage, source-rights, layout or feedback blockers;
- human review of text readability and feedback coherence.

### What remains outside first L1 scope

Out of scope:

- long multi-text comparison;
- two long texts in one capsule;
- full Competencia Lectora readiness claim;
- generated reading texts as final student-facing content;
- productive agentic text/feedback generation;
- broad L1 route-order/continuity implementation;
- public Sales-Ready claim.

### L1 smoke/readiness criteria

Before human UI smoke:

- visible L1 correlativo defined;
- text base readable on desktop/mobile;
- questions do not overload the same screen unnecessarily;
- fragments/citations are clear when used;
- 1-4 answerable questions;
- brief/complete authoredFeedback present;
- feedback cites the same text evidence shown to the student;
- no internal leakage or agentic/commercial claims.

## Cross-program authored standards

M2 and L1 must inherit the accepted PAES standards:

- visible stable correlativo;
- M2 maximum 4 exercises;
- L1 1-4 answerable questions per capsule;
- authoredFeedback brief/complete;
- correctOptionKey reviewed and stable;
- stimulus visible and legible when required;
- table/graph/figure/image/math/text-base dependencies classified before implementation;
- deterministic renderer or approved static asset for final visual stimuli;
- no free-form AI image/figure as final student-facing asset;
- no internal notes, key-check-only feedback or leakage;
- no Sales-Ready, adaptive AI, mastery, theta, PAES score or route-certainty claims;
- static review before human smoke;
- human smoke/readiness before pilot inclusion.

## What remains out of scope

Out of scope for this first M2/L1 minimum scope:

- full M2 or L1 roadmap;
- full PAES coverage;
- public Sales-Ready;
- MVP-Beta cerrado completo;
- productive agentic behavior;
- implementation of capsules;
- registry edits;
- route-order/continuity changes;
- DB/schema/Prisma changes;
- UI changes;
- visual renderer implementation;
- fixture creation or mutation.

## Risk assessment

Key risks:

| risk | impact | mitigation |
| --- | --- | --- |
| M2 becomes nominal, with too little content | pilot promise overstates product breadth | require 4-8 authored M2 capsules before pilot inclusion |
| L1 text-base UI is underestimated | reading skill becomes layout-confusion instead of comprehension | start with 1-2 short text-base capsules and smoke layout early |
| visual stimulus debt reappears in M2 | item validity risk | classify table/graph/figure/math dependencies before implementation |
| M1 patterns are copied blindly | wrong pedagogy for M2/L1 | require separate M2 and L1 seed plans |
| agentic is opened too early | quality and trust risk | keep productive agentic blocked until authored bank and QA thresholds |
| route-order/continuity is changed prematurely | scope expansion | keep continuity implementation in later explicit phases |

## Recommended implementation sequence

Comparison:

| option | assessment |
| --- | --- |
| A. implementar M2 primero | best immediate next implementation family after this scope because M2 extends PAES math promise and can reuse some M1 mental model without copying M1 blindly |
| B. implementar L1 primero | strategically important, but text-base UI/content conventions should be planned after M2 seed scope unless the next pilot specifically prioritizes reading |
| C. planificar ambos y luego implementar uno | this phase does that; next should move to one seed plan |
| D. crear seed plan M2 y seed plan L1 separados | preferred; keeps pedagogy and stimulus risks separate |
| E. volver a M1 antes de M2/L1 | not preferred now because M1 already passed current regression and further M1 growth risks narrow overbuild |

Recommended sequence:

1. `MVP-CONTENT-PAES-M2-AUTHORED-SEED-PLAN-1`
   Plan the first M2 authored seed batch, including topics, item forms, feedback rubric, visual-stimulus classification, and smoke gates.

2. `MVP-CONTENT-PAES-L1-AUTHORED-SEED-PLAN-1`
   Plan the first L1 authored text-base seed batch, including source text types, question counts, feedback rubric and layout/readability gates.

3. `MVP-CONTENT-PAES-M2-AUTHORED-SEED-BATCH-1`
   Implement only after the M2 seed plan passes scope and acceptance gates.

4. `MVP-CONTENT-PAES-L1-AUTHORED-SEED-BATCH-1`
   Implement only after the L1 seed plan passes scope and acceptance gates.

5. `MVP-SALES-PILOT-PAES-M1-M2-L1-CLOSED-SCOPE-PLAN-1`
   Decide whether the next closed pilot includes M2/L1 student exposure or keeps them internal preview.

6. `MVP-GOV-AGENTIC-LAYER-ENTRY-CRITERIA-1`
   Define internal supervisor/QA entry criteria after authored M2/L1 seed planning is underway.
