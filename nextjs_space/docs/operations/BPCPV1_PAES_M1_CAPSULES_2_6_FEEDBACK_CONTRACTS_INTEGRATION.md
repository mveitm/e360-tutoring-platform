## 9. Shared feedback principles across capsules 2-6

Across capsules 2-6, future feedback must:

- acknowledge result without judgment;
- name the mathematical focus;
- explain one central idea;
- provide one concrete review action;
- distinguish correctness from understanding;
- distinguish calculation, interpretation, reading, and strategy when relevant;
- use cautious evidence language;
- remain mobile-readable;
- support review, not diagnosis;
- avoid route decisions or readiness claims.

Feedback must not be only correct/incorrect. It must help the student review what happened and what to check next.

## 12. Cross-capsule cautious evidence language

Allowed posture:

```text
early signal
limited evidence
may suggest
could indicate
conviene revisar
requires more evidence
useful for review, not diagnosis
```

Required distinctions:

- arithmetic slip vs concept issue;
- reading/lookup issue vs content issue;
- wrong operation vs correct idea with calculation mistake;
- representation issue vs interpretation issue;
- magnitude confusion vs formula recall issue;
- self-report mismatch vs performance pattern.

No single item, capsule, or 2-6 completion pattern can close diagnosis or route certainty.

## 13. Cross-capsule mobile readability requirements

Future feedback and step-by-step authoring should respect mobile-first use:

- feedback breve should be short and visible in the flow;
- feedback completo should use short paragraphs or compact blocks;
- step-by-step should use 4-6 short ordered steps;
- notation should be readable without horizontal scrolling where possible;
- each item should separate calculation from interpretation;
- capsule 4 and capsule 6 require special care because each has 8 current-source items;
- explanations should not become generic lessons.

Mobile readability is a contract requirement, not a UI implementation authorization.
