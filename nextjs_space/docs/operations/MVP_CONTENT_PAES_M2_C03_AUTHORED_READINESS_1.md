## Pedagogical purpose

Proposed purpose:

```text
Develop initial interpretation of functional behavior from representations, connecting algebraic rules, tables or controlled textual graph descriptions to changes, intercepts, trends and parameter meaning, without relying on an unaudited visual graph.
```

This purpose continues M2-C01 by deepening the function/representation branch instead of repeating basic expression setup. It also uses the error-control discipline reinforced in M2-C02: identify the relevant representation, choose the correct comparison or feature, and justify the interpretation.

M2-C03 should not claim broad function mastery, graph mastery, PAES readiness, score prediction, adaptive diagnosis or complete PAES_M2 coverage.

## Proposed capsule structure

Recommended M2-C03 structure:

1. Brief introduction focused on interpreting functional behavior across representations.
2. Four multiple-choice exercises.
3. No final student-facing graph unless implementation uses a controlled, auditable representation.
4. Prefer table/textual representation bridge for the first implementation if graph rendering is not explicitly scoped.
5. Brief feedback after answer submission.
6. Complete feedback with step-by-step explanation.
7. Concrete recommendation for improving representation reading or function-behavior interpretation.
8. Autorreporte using the existing capsule flow.

```text
4
```

This respects the M1/M2 maximum of 4 exercises per capsule and chooses the upper target because C01/C02 both closed successfully with 4 exercises.

## Exercise profile proposal

This is a profile for authoring, not final exercise content.

Exercise 1:

```text
Identify a function feature from a simple rule or compact table, such as initial value, fixed change, increasing/decreasing behavior, or output for a given input.
```

Exercise 2:

```text
Match an algebraic rule to a controlled representation described in text or table form, avoiding an unsupported visual graph unless graph rendering is scoped.
```

Exercise 3:

```text
Interpret the effect of changing a parameter, rate, intercept or condition in a simple functional model.
```

Exercise 4:

```text
Solve an integrative PAES_M2-style representation decision where the student must justify which function behavior matches the context and reject a plausible distractor.
```

Guardrails:

- Do not write final stems, options, keys or authoredFeedback in this readiness phase.
- Do not require image assets.
- Do not require a plotted graph unless a later implementation explicitly scopes and validates the graph representation.
- Keep notation plain-text if unambiguous; if notation becomes ambiguous, implementation must define a math-rendering approach before content is accepted.
- Avoid excessive length and keep mobile review readable.

## Feedback authored expectations

Every M2-C03 item must include complete authoredFeedback:

```text
authoredFeedback.briefId
authoredFeedback.completeId
authoredFeedback.version
authoredFeedback.brief
authoredFeedback.complete
```

Brief feedback should:

- identify the key representation feature;
- name the relevant input/output, parameter, trend, intercept, rate or comparison;
- orient the student without turning into full solution text;
- avoid generic correct/incorrect-only feedback.

Complete feedback should:

- restate what the item asks;
- identify the representation used;
- explain the relevant data, rule, table entry, parameter or trend;
- solve or compare step by step;
- justify why the correct option follows;
- explain the likely error behind distractors, especially input/output confusion, intercept/rate confusion, graph-feature misread or overgeneralization from one point;
- connect the item to `Funciones y representacion`;
- provide one concrete improvement recommendation.

Feedback must not include placeholders, internal QA language, score/readiness/mastery claims, Sales-Ready claims, adaptive diagnosis claims or route-certainty claims.
