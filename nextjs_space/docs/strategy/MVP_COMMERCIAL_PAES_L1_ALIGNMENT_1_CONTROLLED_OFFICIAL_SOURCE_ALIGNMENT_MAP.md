# MVP-COMMERCIAL-PAES-L1-ALIGNMENT-1 - Controlled Official-Source Alignment Map

## 1. Phase

* Phase: `MVP-COMMERCIAL-PAES-L1-ALIGNMENT-1`.
* Type: documentation-only / controlled L1 official-source alignment / candidate route-source mapping.
* Product horizon: roadmap change-control / L1 official-source alignment / Sales-Ready support.
* This is not `MVP-Beta-Sales-Ready` approval.
* M2 remains not ready.
* L1 remains not ready.
* Public Sales-Ready remains blocked.

## 2. Relationship to SOURCE-1C, M2-ALIGNMENT-1 and L1-READINESS-1

`MVP-COMMERCIAL-PAES-SOURCE-1C` created a controlled, copyright-safe high-level source-content summary for M2 and L1.

`MVP-COMMERCIAL-PAES-M2-ALIGNMENT-1` created the parallel controlled M2 official-source alignment map and preserved the boundary between high-level mapping, taxonomy, content, implementation, and readiness.

`MVP-COMMERCIAL-L1-READINESS-1` versioned a non-binding candidate L1 route/content preparation proposal.

This document maps the L1-READINESS-1 candidate L1 route and StudyLoad families against the high-level official L1 source summaries already extracted in SOURCE-1C.

It does not re-extract official sources. It does not replace SOURCE-1C. It does not validate the L1 proposal as final.

## 3. Nature of document

This is a controlled alignment map.

It is:

* documentation-only;
* high-level;
* source-summary based;
* conservative;
* intended to support later L1 taxonomy, text-rights, editorial-quality, and route-readiness phases.

It is not:

* full source extraction;
* official table reproduction;
* official text reproduction;
* official question reproduction;
* final L1 taxonomy;
* L1 content design;
* text selection;
* question design;
* StudyLoad design;
* feedback design;
* implementation authority;
* L1 readiness approval;
* Sales-Ready approval.

## 4. Purpose

The purpose is to identify where the candidate L1 proposal appears compatible with the high-level official L1 source summary and where it remains too broad, vague, or unsupported for taxonomy or implementation.

This phase helps future work decide:

* which candidate route families have high-level source fit;
* which candidate route families need deeper source-content extraction;
* which candidate StudyLoad families are only conceptual;
* which text rights and editorial-quality dependencies must be resolved before content;
* which risks must remain reserved for Pro/high-level direction;
* which future phases must happen before L1 taxonomy, texts, questions, StudyLoads, feedback, dashboard evidence, or readiness can be considered.

## 7. Alignment method

The mapping method is intentionally limited:

1. Start from the high-level SOURCE-1C L1 reading-skill families and text/content orientation.
2. Compare each candidate L1 route family to those high-level categories only.
3. Mark high-level source fit using conservative language.
4. Identify gaps, overreach risks, text-rights/editorial dependencies, and deeper extraction needs.
5. Preserve a hard boundary between candidate alignment and final taxonomy.

Alignment language means only candidate compatibility at a high level. It does not mean the proposal is officially aligned, complete, implementation-ready, or ready for students.

## 8. Official high-level L1 reading-skill families

SOURCE-1C summarized the official high-level L1 reading-skill families as:

* locating information;
* interpreting meaning and relationships;
* evaluating text, context, form, purpose, and position.

This document does not infer lower-level subskills under those families.

## 9. Official high-level L1 text/content orientation

SOURCE-1C summarized the official high-level L1 text/content orientation as:

* reading tasks are tied to texts;
* text selection includes literary and non-literary contexts;
* text formats may vary;
* questions are multiple-choice and text-referenced;
* knowledge is treated as support for reading tasks, not as standalone conceptual recall.

This document does not infer detailed text types, official passage categories, item designs, or question formats beyond that summary.

## 12. Aligned areas

The candidate proposal appears compatible at a high level in these areas:

* locate evidence maps naturally to `locating information` and text-referenced reading tasks;
* controlled inference maps plausibly to `interpreting meaning and relationships`;
* purpose and viewpoint maps plausibly to `evaluating text, context, form, purpose, and position`;
* contextual vocabulary appears compatible only when treated as meaning-in-context work tied to a text;
* relationship between ideas and structure appears compatible with interpretation and possibly evaluation of form/structure;
* integrated questions may be useful later if constrained by source-aligned reading skills and text conditions.

These are candidate alignment observations only. They do not establish official alignment, content readiness, text approval, question approval, or taxonomy completeness.

## 13. Gaps and weakly supported areas

Current gaps:

* Literary and non-literary text handling is not yet mapped to candidate route families.
* Variable text formats are not yet mapped to candidate route families.
* Text rights, licensing, original-text policy, adaptation policy, and editorial review are unresolved.
* Distractor review is likely a feedback/review family, not necessarily a source-derived route family.
* `integrated questions` is too broad without source-content detail and taxonomy criteria.
* `initial reading diagnostic` is not source-defined yet and needs product/pedagogical direction before being treated as a route component.
* Candidate StudyLoad families are conceptual and do not include source-backed objectives, text criteria, item criteria, feedback boundaries, or dashboard evidence.

## 14. Overreach risks

Risks:

* Treating this map as final official L1 alignment.
* Treating high-level SOURCE-1C summaries as enough to define a detailed taxonomy.
* Treating L1 as a generic multiple-choice quiz.
* Treating `integrated questions` as a catch-all for unsupported content.
* Treating contextual vocabulary as standalone word-definition drills.
* Treating distractor review as a source-derived route family rather than a feedback/review need.
* Creating L1 texts, questions, or StudyLoads before source-content extraction, text rights, editorial review, taxonomy, and feedback design.
* Declaring L1 ready because high-level categories appear compatible.
* Using dashboard/progress or feedback language before L1 content exists.

## 15. Text rights / editorial-quality dependencies

L1 has text-rights and editorial-quality dependencies that M2 does not have in the same way.

Future L1 content work must separately resolve:

* whether texts are original, licensed, public-domain, adapted, or externally sourced;
* whether any official text, released item, example text, or passage may be used at all;
* rights/licensing conditions for every non-original text;
* difficulty, clarity, ambiguity, sensitivity, bias, and age-appropriateness;
* cognitive load and text length;
* how feedback may reference textual evidence without copying excessive text;
* how to store text and question metadata safely;
* editorial review thresholds before student use.

This phase approves none of those dependencies.

## 17. L1 taxonomy boundary

This document does not create a L1 taxonomy.

A future L1 taxonomy phase must be separate and must depend on:

* accepted source-content extraction detail;
* approved text-rights and editorial-quality boundaries;
* an approved mapping method;
* explicit inclusion/exclusion criteria;
* separation from generic quiz behavior;
* Pro/high-level direction on granularity and readiness threshold;
* later content, feedback, dashboard, and app/student evidence phases.

The alignment map is an input to later taxonomy work, not the taxonomy itself.

## 18. Future L1 source-content needs

Future L1 source-content work should:

* cite the official L1 page and PDF exactly;
* extract only necessary official structure in a copyright-safe way;
* avoid copying official tables;
* avoid copying official texts, passages, examples, questions, item wording, or answer keys;
* identify official source detail needed to distinguish L1 from generic reading quizzes;
* clarify how literary and non-literary contexts should affect route taxonomy;
* clarify how variable text formats should affect route taxonomy;
* clarify how locating, interpreting, and evaluating families should be represented without inventing subskills;
* define which official details are safe to store and paraphrase.

## 19. Future L1 route taxonomy prerequisites

Before a L1 route taxonomy phase:

* source-content detail must be sufficient to support route boundaries;
* candidate route families must be reviewed for coverage gaps;
* text-rights and editorial-quality constraints must be defined;
* `integrated questions` must be constrained;
* distractor review must be classified as route, feedback, or review behavior;
* contextual vocabulary must be constrained as text-bound meaning work unless later alignment says otherwise;
* diagnostic logic must be scoped;
* initial route depth and breadth must be reserved for Pro/high-level direction.

## 20. Future L1 feedback/dashboard implications

Future feedback implications:

* L1 feedback must not be generic answer-key feedback.
* Feedback should later be text-evidence aware without copying excessive text.
* Feedback should connect the question, the text, and the reading skill when supported.
* Distractor review needs future item-level and feedback-level design.
* Ambiguity and editorial-quality review are critical before feedback can be trusted.

Future dashboard implications:

* L1 progress must be separate from M1 and M2 progress.
* Dashboard evidence must rely on real L1 reading activity, not labels.
* Progress categories must be reading-specific and supported by actual text/question evidence.
* Progress categories must not imply PAES score, theta, mastery, ranking, guaranteed improvement, or complete preparation.
* App/student evidence requires separate implementation and validation phases.

## 21. PASS / FAIL / BLOCKED interpretation

Documentation PASS for this phase means:

* the candidate L1 proposal has been mapped conservatively to SOURCE-1C high-level categories;
* gaps, overreach risks, text-rights dependencies, and editorial dependencies are documented;
* taxonomy, content, texts, questions, StudyLoads, feedback, dashboard, implementation, readiness, and Sales-Ready boundaries remain intact.

This phase would FAIL if it:

* creates final taxonomy;
* creates L1 content;
* creates texts;
* creates questions;
* creates StudyLoads;
* creates registry entries;
* copies official tables;
* copies official texts;
* copies official questions;
* declares L1 ready;
* approves Sales-Ready.

Future L1 readiness remains BLOCKED until separate phases produce source-aligned taxonomy, text-rights/editorial policy, content/text/question design, feedback, dashboard evidence, app/student evidence, and required governance approvals.

## 22. Risks and mitigations

| Risk | Why it matters | Mitigation |
| --- | --- | --- |
| High-level mapping becomes hidden taxonomy | Would bypass source detail and Pro/high-level direction | State taxonomy boundary and require future taxonomy phase |
| Source summaries are overread | SOURCE-1C intentionally avoids detailed tables and texts | Use only high-level reading-skill and orientation labels and mark gaps |
| Candidate StudyLoad labels become implementation scope | Could create registry, texts, questions, or content before readiness | Label all StudyLoad families as conceptual only |
| L1 becomes a generic quiz | Breaks L1 identity and Sales-Ready gate expectations | Require text-tied reading skills, feedback identity, and editorial review |
| Text rights are ignored | Creates legal, ethical, and brand risk | Require separate text-rights and editorial-quality phases |
| Contextual vocabulary becomes standalone drill | Could drift from text-based reading task orientation | Constrain vocabulary as meaning-in-context work |
| Integrated questions become a vague bucket | Weakens route quality and evidence | Require constraints before taxonomy or content |

## 23. Questions reserved for Pro/high-level direction

Reserved questions:

* Exact L1 taxonomy granularity.
* Whether literary and non-literary contexts need explicit initial route families.
* Whether variable text formats need explicit initial route families.
* Whether initial diagnostic work should be visible, dynamic, or both.
* Minimum L1 coverage before controlled sandbox use.
* Minimum L1 coverage before any public Sales-Ready consideration.
* Text sourcing, creation, adaptation, and licensing policy.
* Whether official released/example texts or questions may ever be used internally.
* Required editorial, sensitivity, ambiguity, and cognitive-load review threshold before L1 content creation.
* Whether integrated questions should be a late route family or only an assessment/review mode.
* Required app/student evidence before L1 can count toward a public offer.
